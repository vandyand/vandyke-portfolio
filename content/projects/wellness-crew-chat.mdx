## Problem

A single do-everything chatbot gives generic answers; a hard-coded
keyword router gives brittle ones. The demo problem: route each user
message to the *right* specialist persona, with enough shared context
that the specialists feel like a team rather than four separate bots.

## What I built

A chat interface backed by five agents. A **router agent** analyzes
every incoming message against the conversation history and returns a
structured `RoutingDecision` — target character, confidence, and
reasoning — via strict JSON schema output. The chosen specialist then
responds in persona:

- **Coach** — fitness and exercise, energetic and motivational
- **Chef** — nutrition and diet, warm and practical
- **Sage** — stress and mindfulness, calm and philosophical
- **Doc** — general health, evidence-based and direct

All four share the running conversation memory, so characters build on
each other's answers, and an internal character-notes system tracks
user preferences across the session.

## Outcome

Ask about meal prep, then stress, then training — the router hands you
off seamlessly and each character knows what the others already said.
A compact demonstration of the routing-plus-shared-state pattern behind
production multi-persona assistants.
