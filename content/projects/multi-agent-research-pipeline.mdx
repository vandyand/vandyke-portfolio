## Problem

Chaining LLM agents with free-text handoffs means every step re-parses
the previous step's prose and silently degrades. The interesting
engineering question isn't "can agents talk" — it's whether the
*contract* between agents can be strict enough that their outputs
compose like function calls.

## What I built

A pipeline of three agents, each built as a **Schemaon** — an atomic
reasoning primitive with a defined JSON output schema enforced by the
LLM's strict structured-output mode:

1. **Researcher** analyzes the topic and produces structured findings,
   each with a relevance score.
2. **Writer** drafts a multi-section article directly from the findings
   object — the Researcher's output *is* the Writer's input, no glue
   code.
3. **Critic** evaluates the draft on clarity, accuracy, and engagement,
   returning a structured critique.
4. **Writer (revision)** receives the critique appended to its
   conversation history and produces an improved article against the
   same output schema.

Every response is validated against its schema before the next agent
runs. The revision loop is the key demonstration: multi-turn
coordination where the same agent, same schema, and richer context
yield a measurably tightened second draft.

## Outcome

Enter a topic, watch four agent passes stream in: findings, draft,
critique, revision. Because the seams are schemas rather than prompts,
any agent can be swapped or upgraded independently — the same property
that makes production multi-agent systems maintainable instead of
haunted.
